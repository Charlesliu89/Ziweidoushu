## 规则取舍

主引擎继续使用 `iztro`，因为它和文墨样例在农历、四柱、命身宫、主辅星、生年四化、大限、流年、流月能力上已经对齐。

文墨风格差异主要不在底层排盘，而在展示和叠加规则：

- 宫名口径：`仆役` 显示为 `交友宫`。
- 生年四化：显示为 `生年禄/权/科/忌`，而不是单独的 `禄/权/科/忌`。
- 宫干自化：本宫宫干四化命中本宫星曜，显示 `↓禄/↓权/↓科/↓忌`。
- 向心自化：对宫宫干四化命中本宫星曜，显示 `↑禄/↑权/↑科/↑忌`。
- 来因宫：使用 `iztro` 的 `isOriginalPalace`，在盘面标记 `来因`。
- 大限/流年/流月四化：按对应天干查十干四化表生成，不用重新实现安星算法。

## 当前项目结构

当前仓库只保留可上传的排盘核心、文墨标记规则、演示验证脚本和技术文档：

- `src/cli.ts`：命令行入口，只把结果写到用户指定的本地路径。
- `src/index.ts`：库入口。
- `src/lib/astro.ts`：调用 `iztro` 生成基础命盘。
- `src/lib/serialize.ts`：把排盘结果序列化为稳定 JSON 结构。
- `src/lib/wenmo-rules.ts`：生成文墨展示标记和流限摘要。
- `scripts/validate-demo-chart.mjs`：使用演示参数验证核心文墨标记，不保存真实案例。
- `scripts/privacy-audit.mjs`：上传前检查 `.gitignore` 和公开文件中的隐私风险。
- `docs/`：只放技术说明，不放真实输入、命盘 JSON 或解读记录。

真实输入、排出的盘、导出报告和案例记录必须留在 `.gitignore` 覆盖的本地目录，例如 `output/`、`inputs/`、`private/`、`exports/`。

主要公开函数：

- `buildWenmoPalaceDecorations(chart)`：为十二宫生成文墨展示标记。
- `getStemSihuaLabels(stem)`：按天干输出四化星。
- `toFullSihuaName(value)`：把 `iztro` 的 `禄/权/科/忌` 统一成 `化禄/化权/化科/化忌`。
- `summarizeWenmoHoroscope(chart, date)`：整理大限、流年、流月摘要。

## 样例校验目标

对演示参数 `2000-01-01 00:00 男`，规则层应补出文墨样例中的关键标记：

- 福德宫左辅：`↑科`
- 官禄宫太阴：`↓权`
- 官禄宫文曲：`生年忌`
- 交友宫贪狼：`生年权`、`↓权`
- 交友宫：`来因`
- 财帛宫天同：`↑禄`
- 财帛宫天梁：`生年科`、`↓禄`
- 子女宫武曲：`生年禄`
- 夫妻宫太阳：`↓忌`
- 命宫天机：`↓权`
